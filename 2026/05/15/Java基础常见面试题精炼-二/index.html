Java 基础面试题精炼（二）：面向对象与异常处理

长星拓2026-05-152026-06-12

前言

本文精炼 Java 面向对象与异常处理 共 10 道高频面试题，聚焦 OOP 核心概念和语法特性的面试回答思路。

一、String 底层实现是怎样的？

JDK 版本不同，底层实现不同：

版本 底层结构 编码方式
JDK 8 及之前 char[] UTF-16，每字符 2 字节
JDK 9 之后 byte[] Latin1（1字节）或 UTF-16（2字节）

JDK 9 优化原理：

  • 如果字符串所有字符都是 Latin1 字符集（英文等西欧语言）→ 每字符 1 字节，节省内存
  • 如果包含中文、韩文、emoji 等 → 每字符 2 或 4 字节，UTF-16 编码

面试答法：JDK 9 把 char[] 改成 byte[]，根据内容自动选择编码，拉丁字符省内存。

二、普通 for 和增强 for 的区别

适用场景和使用方式不同：

特性 普通 for 增强 for
适用范围 所有循环场景 仅数组和 Collection
下标访问 ✅ 支持 ❌ 不支持
修改元素 ✅ 可以 ❌ 不可以（只读）
底层实现 直接循环 数组→普通for，集合→Iterator

面试关键点：

  • 增强 for 是语法糖，编译后会转换
  • 需要修改元素或使用下标时，必须用普通 for
  • 遍历集合时用增强 for 更简洁

三、方法重载与重写的区别

重载（Overload）在同一个类，重写（Override）在父子类：

维度 重载（Overload） 重写（Override）
位置 同一个类中 子类重写父类方法
方法名 必须相同 必须相同
参数列表 必须不同 必须相同
返回值 无要求 相同或协变
访问修饰符 无要求 不能比父类更严格
绑定时机 编译时（静态分派） 运行时（动态分派）

面试答法：重载是”同一个类，同名不同参”；重写是”子类覆盖父类同名同参方法”。

四、final 关键字的作用

final 可修饰变量、方法、类，作用各不相同：

修饰对象 效果 示例
变量 常量，初始化后不能修改 final int MAX = 100;
方法 不能被子类重写 public final void show() {}
类 不能被继承 public final class String {}

面试补充：

  • final 修饰的引用类型，地址不能变，但对象内容可以变
  • final List<String> list = new ArrayList<>(); → 不能重新赋值，但可以 add/remove

五、抽象类与接口的区别

核心区别：抽象类是”is-a”关系，接口是”can-do”能力。

维度 抽象类 接口
继承/实现 单继承 多实现
构造方法 ✅ 有 ❌ 没有
成员变量 任意类型 默认 public static final
方法 抽象方法 + 普通方法 抽象方法 + default + static
设计意图 定义公共行为和状态 定义行为规范（能力）

JDK 8 接口新特性：

  • default 方法：有默认实现，子类可选择重写
  • static 方法：通过接口名调用

面试答法：抽象类用于代码复用（模板模式），接口用于定义契约（策略模式、多态）。

六、面向对象三大特性的理解

封装、继承、多态，三者层层递进：

特性 核心思想 Java 实现
封装 隐藏内部细节，只暴露必要接口 private + getter/setter
继承 子类复用父类代码 extends 关键字
多态 同一接口，不同实现 方法重写 + 向上转型

封装的好处：保护数据安全，提高可维护性

继承的好处：代码复用，建立类层次结构

多态的好处：灵活性和扩展性，符合开闭原则

面试答法：封装保护数据，继承复用代码，多态提升扩展性。

七、访问修饰符的作用范围

四种修饰符，权限由大到小：

修饰符 本类 本包 其他包子类 其他包非子类
public ✅ ✅ ✅ ✅
protected ✅ ✅ ✅ ❌
默认（缺省） ✅ ✅ ❌ ❌
private ✅ ❌ ❌ ❌

可以修饰的对象：

修饰符 外部类 成员变量 成员方法 内部类
public ✅ ✅ ✅ ✅
protected ❌ ✅ ✅ ✅
默认 ✅ ✅ ✅ ✅
private ❌ ✅ ✅ ✅

面试答法：外部类只能用 public 或默认；成员可以用任意修饰符。

八、throw 与 throws 的区别

throw 是抛出异常对象，throws 是声明可能抛出的异常类型：

维度 throw throws
位置 方法体内部 方法声明上
作用 抛出具体异常对象 声明异常类型
后跟内容 异常实例（如 new RuntimeException()） 异常类名（如 IOException）
执行效果 立即终止方法，异常抛出 告诉调用者需处理该异常

面试答法：throw 是”我抛出一个异常”，throws 是”这个方法可能抛出异常，请调用者处理”。

九、Java 的集合容器都有哪些？（高频）

集合框架两大体系：Collection 和 Map。

Collection 体系：

接口 实现类 特点
List ArrayList、LinkedList、Vector 有序、可重复
Set HashSet、LinkedHashSet、TreeSet 无序、不可重复
Queue PriorityQueue、ArrayDeque 队列结构

Map 体系：

实现类 特点
HashMap 最常用，无序，key 可 null
LinkedHashMap 保持插入顺序
TreeMap 按 key 排序
Hashtable 线程安全，key 不能 null
ConcurrentHashMap 线程安全，高并发首选

面试答法：List 可重复有序，Set 不可重复无序，Map 存键值对。常用选 ArrayList、HashSet、HashMap。

十、Collection 和 Collections 有什么区别？

Collection 是接口，Collections 是工具类：

类型 说明
Collection 单列集合的顶层接口，定义了 List、Set、Queue 的通用方法
Collections 集合工具类，提供排序、线程安全包装、查找等静态方法

Collections 常用方法：

  • Collections.sort(list) — 排序
  • Collections.synchronizedList(list) — 转线程安全
  • Collections.unmodifiableList(list) — 转不可修改
  • Collections.max(list) / min(list) — 获取最大最小值

面试答法：Collection 是集合家族的”族长”（接口），Collections 是集合的”管家”（工具类）。

结语

这 10 道题覆盖了 Java 面向对象的核心：String 底层、循环对比、重载重写、final、抽象类接口、OOP 特性、访问修饰符、异常处理、集合体系。面试时关键是把概念区分 + 对比维度说清楚。
