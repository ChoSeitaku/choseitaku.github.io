Java 基础面试题精炼（一）：核心概念与基础语法

长星拓2025-05-142026-06-12

前言

本文精炼 Java 基础 核心概念与基础语法共 10 道高频面试题，每道题聚焦面试回答的核心思路和关键结论，帮助快速掌握面试中的标准答法。

一、JDK 和 JRE 有什么区别？

一句话：JRE 是运行环境，JDK 是开发工具包（JRE + 开发工具）。

组件 说明
JRE Java 运行环境，包含 JVM + 核心类库，只能运行 Java 程序
JDK Java 开发工具包，包含 JRE + 开发工具（javac、java、jconsole 等）

面试答法：JDK = JRE + 编译器 + 调试工具。开发用 JDK，部署只需要 JRE。

二、& 和 && 有什么区别？

核心区别：& 是按位与（两边都计算），&& 是短路与（左边 false 右边不计算）。

运算符 适用类型 执行特点
& 数值（按位与）、boolean 左右两边都会计算
&& 仅 boolean 左边为 false 时，右边不计算（短路）

实际开发中：判断条件几乎都用 &&，避免不必要的计算和潜在的空指针异常。

三、return、continue、break 的区别

三个关键字的作用范围和效果完全不同：

关键字 作用范围 效果
break 循环体 / switch-case 结束本层循环或 switch 结构
continue 循环体 跳过本次循环剩余语句，进入下一次循环
return 方法体 结束当前方法，返回调用处

面试补充：嵌套循环中，break 只跳出当前层，要跳出多层需要使用标签。

四、== 和 equals 的区别（高频）

核心区别：== 比较地址值（引用类型）/ 数值（基本类型），equals 默认也比较地址，但可重写比较内容。

比较方式 基本类型 引用类型
== 比较数值 比较内存地址
equals 不适用 默认比较地址，重写后比较内容

面试关键点：

  • equals 是 Object 类的方法，只能通过对象调用
  • String、Integer 等类已重写 equals，比较的是内容
  • 自定义类需手动重写 equals 才能比较属性

五、hashCode 和 equals 的关系（高频）

不是！hashCode 相同，equals 不一定为 true。

场景 hashCode equals
equals 为 true 一定相同 true
hashCode 相同 相同 不一定为 true（哈希冲突）

面试关键点：

  • hashCode 和 equals 是两个独立方法，可以分别重写
  • hashCode 相同只说明在同一个哈希桶，不代表对象相等
  • 重写 equals 必须重写 hashCode，否则放入 HashMap/HashSet 会出问题

六、int 和 Integer、BigInteger 有什么区别？

三种类型分别适用于不同场景：

类型 类别 范围 特点
int 基本类型 ±21亿 性能最高，无对象开销
Integer 包装类 ±21亿 可为 null，支持泛型和集合
BigInteger 不可变大整数 理论无限 支持任意精度运算

类似考点：double、Double、BigDecimal 的区别同理。

面试答法：普通运算用 int，需要对象特性用 Integer，超大数运算用 BigInteger。

七、String 是基本数据类型吗？

不是！String 是引用数据类型。

Java 8 种基本数据类型：byte、short、int、long、float、double、char、boolean。

String 是类，存储在堆内存中，变量存储的是引用（地址）。

八、字符串操作类有哪些？区别是什么？（高频）

三个核心类：String、StringBuilder、 StringBuffer。

特性 String StringBuilder StringBuffer
可变性 不可变 可变 可变
线程安全 安全（不可变） 不安全 安全（synchronized）
性能 拼接慢 最快 较快
适用场景 少量操作 单线程大量拼接 多线程大量拼接

面试关键点：

  • String 不可变指常量池中的字符串对象不可变，变量引用可以指向新对象
  • 频繁拼接用 StringBuilder（如循环中拼接 SQL）
  • 多线程共享拼接用 StringBuffer

九、String str=”abc” 和 String str=new String(“abc”) 的区别？

创建对象数量不同：

写法 创建对象 内存位置
String str = "abc" 1 个（常量池） 常量池
String str = new String("abc") 2 个（常量池 + 堆） 堆 + 常量池

详细说明：

  • 字面量方式：直接在常量池创建 “abc”，str 指向常量池
  • new 方式：常量池创建 “abc”，堆中创建 new String 对象，str 指向堆

面试补充：String.intern() 方法可以将堆中的字符串引用转为常量池中的引用。

十、String 类 isEmpty 和 isBlank 的区别？

判断”空”的标准不同：

方法 判断标准 示例
isEmpty() 长度是否为 0 "" → true，" " → false
isBlank() 是否为空或全空白字符 "" → true，" \t\n" → true

代码验证：

面试答法：isEmpty 只看长度，isBlank 还会忽略空格、制表符、换行符等空白字符。

结语

这 10 道题覆盖了 Java 基础的核心概念：JDK/JRE、运算符、流程控制、比较运算、包装类、String 核心考点。面试时关键是把核心区别 + 对比维度说清楚。
